# Contributing to cryptodiff

Thanks for your interest in `cryptodiff`.

This repository is public for transparency and adoption, but the project is not currently managed as an open-source codebase.

## Current contribution policy

- Issues and feature requests are welcome.
- Bug reports with reproduction steps are welcome.
- Security reports should be sent privately (do not open a public issue for sensitive details).
- Unsolicited pull requests are not currently accepted by default.

## Why pull requests are currently limited

`cryptodiff` is part of a broader commercial product strategy.
To keep roadmap, architecture, and release commitments coherent, code changes are currently maintained by the core team.

We may invite or accept targeted external contributions in the future under explicit terms.

## How to submit useful issues

Please include:

- `cryptodiff` version or commit SHA
- OS and runtime details
- Exact command used
- Relevant config/policy snippets (redacted if needed)
- Expected behavior vs actual behavior
- Reproduction steps and sample files where possible

## Security vulnerabilities

If you believe you found a security issue, please do not disclose it publicly first.
Follow the private disclosure process in `SECURITY.md`.
